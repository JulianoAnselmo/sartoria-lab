# Sartoria Lab Site v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Sartoria Lab institutional site (home, ofício, peças, medida, caderno) as a static-first Astro 4 project with Tailwind and Markdown content collections, deployed to Vercel.

**Architecture:** Static-first Astro project. All content routes are pre-rendered. Only the `/medida` form POSTs to a serverless endpoint (Formspree by default). Brand tokens (paleta Sartoria Clássica + fontes Google) live in `tailwind.config.mjs` and `src/styles/fonts.css`. Cases and caderno posts live under `src/content/` as Markdown with typed frontmatter.

**Tech Stack:** Astro 4, Tailwind CSS, TypeScript, Zod (via content collections), @astrojs/sitemap, @astrojs/rss, Vitest (for the form validation logic), Formspree, Vercel.

**Spec:** `docs/superpowers/specs/2026-04-22-site-design.md`
